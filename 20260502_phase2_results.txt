----------------------------------------------------------------
四个核心发现
----------------------------------------------------------------

[1] L11#2089 是隐藏的最强因果节点 (反 attribution 直觉)

  attribution 排第 3 (-2.34), 但 invert 效应排第 1 (+43.6%),
  比 attribution top-1 的 L09#357 (-41.2%) 还强.

  含义: attribution 强度 ≠ 因果强度. L11 作为 mid amplifier,
  虽然 attribution gradient 只有中等大小, 但因为 它 直接喂给
  L12 (距离近, 衰减少), 实际因果权重最大.

  这是 amr_wtf 项目里第二次出现 "attribution top-1 ≠ causal top-1"
  现象 (上次是 LC233 反证里 L09#6039 attribution 高但 invert
  只 -4.6%). 给 attribution-based 筛选打上明确的 上限标记.

[2] L09#6039 FFI motif 实证确认

  在 L33#9054 metric 上 (GB01 attribution + verify):
    attr = +16, rel_ablate = -2.2%, rel_invert = -4.6%
    → driver of L33

  在 L12#4638 metric 上 (本次):
    attr = -2.33, rel_ablate = +4.8%, rel_invert = +9.6%
    → suppressor of L12

  两次实验方向完全一致: L09#6039 同时
    - 直接驱动 L33#9054 (终端)
    - 直接压制 L12#4638 (L33 的 suppressor)
    = 双重正向贡献于 L33#9054 的最终输出

  这是教科书级前馈抑制 (Feed-Forward Inhibition) motif.
  amr_wtf 项目里第一个跨 metric 实证的 motif.

[3] L07#2800 几乎无效 → 早期信号被中间层重塑

  attribution +2.09 (top-5), 但 invert 只 -2.1%, 接近噪声.
  L07 距离 L12 之间还有 4 层 MLP + attention 转换, 信号被
  反复 norm + project, 到 L12 时已与原始 L07 激活方向无关.

  含义: 早期层 attribution gradient 经过的链很长, 即使数学上
  attribution 不为零, 实际因果效应被中间层吸收. 这与之前
  ANALYSIS 里 "L09 与 L28 反序" 的层加权效应同一原因.

  实践: probe 早期层 (L0-L8) 用 region/blockwise diff 没意义,
  必须用 attribution patching 直接绕过中间层验证因果.

[4] Sign consistency 100% (所有 5 节点)

  ablate / amplify / invert 三项符号都和 attribution 符号一致:
    suppressor (attr<0) → ablate+, amplify-, invert+
    driver    (attr>0) → ablate-, amplify+, invert-

  这是 amr_wtf 第二次 (5/1 GB01 verify 也是 100%) 跨 metric
  确认 attribution sign 可作因果方向直接使用.

Part 2 — 链路逐节点 ablation (找 stop signal carrier)
================================================================

代码:    J:/amr/amr_wtf/circuit_chain_per_node.py
日志:    outputs/circuit_attr/circuit_chain_per_node.log
数据:    outputs/circuit_attr/circuit_chain_per_node.pt

预期假设:
  5/1 LC233 反证显示链路全部 ablate 触发复读 (gb01_ablate
  触顶 12288 token, thought 段大段重复). 推测链路里 至少
  1 个节点参与 "决定何时停止生成". 找出来这个节点是 amr_wtf
  里第一个具有可解释行为的 carrier.

设计:
  跑 GB01-B 完整序列 (1821 tokens, 末尾就是 <turn|> EOS=106).
  在 predict_pos = 1819 (倒数第二位, 它的 logit 预测最末 <turn|>)
  上看 logit[<turn|>] 和 prob[<turn|>] 在每种 ablate 下的变化.

  预期: 某个节点 ablate 让 EOS logit 大幅下降 → stop signal carrier

---- 实测结果 (反预期) ----

  baseline 状态:
    EOS logit = +20.875
    EOS prob  =  0.9996       <─── 已经 99.96% 饱和
    TOP1     = <turn|> @ prob 0.9996

  所有 7 节点单 ablate:
    EOS logit Δ ∈ [-0.375, +0.125]
    EOS prob 全部保持 0.9996, 完全不变
    TOP1 始终是 <turn|>, prob 始终 ≈ 1.0

  完整链路 (7 节点同时) ablate:
    EOS logit Δ = +0.000, prob 不变

----------------------------------------------------------------
反结果的解读 (这本身是发现)
----------------------------------------------------------------

[1] EOS 在已写完 response 末尾位置已饱和

  在 GB01-B 的末尾, 模型已经看到:
    - 完整 thought 段
    - 完整 response 段 (中文解题 + C++ 代码 + ``` 反引号收尾)
    - 紧跟 \n (id 107)

  在这个上下文下, "该停止了" 的判断信号已被 attention 从
  ``` token + \n + 用户 turn 结束模式 等多个外部 cue 累积
  推到饱和 (prob=0.9996, logit=+20.875). 任何 MLP 链路扰动
  都没法把它从 99.96% 拉下来.

[2] Stop signal 是 generate-time 累积调制, 不是离散决策

  这反过来印证 5/1 LC233 反证: 之所以 链路 ablate 引发复读,
  是因为在 generate 早期 (thought 中段、response 段早期),
  模型还没看到代码结束符号, EOS 决策依赖链路在每一步累积
  的"该往哪走"信号. 链路被切, 模型不知道 thought 该多长、
  response 段该写多详细, 走到一个 它从未训练过 的状态,
  引发复读.

  本质: stop signal 不能用 single-token logit prob 测,
  必须用 generate-trajectory 测.

[3] 该 metric 设计错误的根本原因

  我选的 metric "predict <turn|> at end of complete response"
  测的是 "看到完整答案后是否同意结束". 但 LC233 反证里我们
  看到的是 "在 generate 中段, 模型还能不能走向自然结束".
  两者完全不同问题.

  正确的 metric 应该是:
    - generate-based: max_new_tokens=12288, ablate per-node,
      看哪个 ablate 让生成长度异常 / 触发复读
    - logprob-based: 在 thought 段中段位置看 EOS 累积 prob 升降

  generate-based 是金标准但慢 (~5-10 分钟/次, 8 次 ~1 小时).
  logprob-based 快但需要慎选位置 (anchor 中段而不是末尾).
