## 0. 实验设置（必读）

- **基模型**：`Gemma 4 E2B-it`（INT8 量化加载，bitsandbytes，35 层 decoder）
- **采样位置**：`model.model.language_model.layers[li].mlp.down_proj` 的输入张量（即 `act(gate) ⊙ up` 后的中间激活；标准的 gated MLP「神经元活动」）
- **差分定义**：`diff = mean_B - mean_A`
  - A = Gemma 自己回答
  - B = 参考模型回答（文学域=Claude Opus 4.6；代码域=Gemini 3 Flash Preview Thinking）
  - 答案 token 对齐到 `min(len(A), len(B))` 后取窗口均值
- **clamp / steering 机制**：
  - `forward_pre_hook` 在 `down_proj` 输入处对指定 neuron 加偏移：`x[:,:,n] += alpha * gain`
  - `gain` = 该神经元固有方向（来自 diff 符号），`alpha` = 全局倍率（UI 可调）
- **关键脚本**：
  - 单 QA 差分：`probe_single.py` / `probe_code.py`
  - 多 QA：`multi_qa_probe.py`
  - 区域化（标注后）：`region_probe.py` + `region_analysis.py`
  - 代码 vs 文学对照：`code_vs_lit_compare.py`
  - 实时实验 UI：`chat/app.py` + `chat/neurons.json`
- **数据**：
  - 文学域：`data/claudeopusQA01..05.json`（5 个 QA，含 `conclusion_analysis` + `glue_sentences`）
  - 代码域：`data/gemini_3_flash_preview_code_qa_01.json`（n=1，HTML/Tailwind 还原 ChatGPT 旧界面）

## 4. 噪声层（明确不要 clamp）

`probe_code.py` global top-30 中 17/30 来自 L00–L04，例：

```
L01#169   diff=-7.99  (lit|d|=0.004)
L01#5195  diff=-6.18  (lit|d|=0.011)
L01#550   diff=+5.50  (lit|d|=0.011)
L00#1928  diff=+3.17  (lit|d|=0.022)
```

- 这些是「token 表面差异」造成的早期层激活差，跟内容功能无关。
- HTML/Tailwind class 名 vs Gemma 自己的 inline style 在 token 层就完全不同，所以 L0–L4 必然出现巨幅差。
- **始终把 clamp 候选限制在 L≥15**（已写入 `pick_L15_persistent` 阈值）。

## 9. 实机验证（2026-04-27 chat UI 多组实验）

任务：复刻 chat.openai.com (GPT-3.5 时代) 静态页面
环境：Tkinter chat UI + clamp hooks，温度=0 贪心生成

### 排序（最佳 → 最差）

| 排名 | 配置 | Think | 关键观察 |
|---|---|---|---|
| 🥇 | **code_v1_minimal** | OFF | **左侧历史栏 + 新建聊天按钮 + 真实可用对话页面 + 模拟历史 + 真实输入区**。最接近原版结构 |
| 🥈 | **code_full** | OFF | **唯一给出 HTML+CSS+JS 三件套**，可以发送消息；仅输入框微小色差 |
| 🥉 | **code_kill_lazy_only**（仅 L16#1298） | OFF | HTML+CSS+JS 三合一；UI 无问题；但风格仍是 Gemma 默认蓝白 |
| 4 | code_v2_extended | OFF | 与其他 V1 等价或略弱，未呈现独立增益 |
| 5 | 其他 lit 系预设 | OFF | 蓝白普通对话 UI，结构远不及代码系 |
| ⚠️ | code_v1 | ON | 仅一个输入框 + 发送按钮 + 标题，结构崩塌 |
| ⚠️ | code_full | ON | 五彩斑斓但无对话区，仅免责声明类信息 |
| ❌ | code_kill_lazy_only | ON | **CSS 直接消失**，仅给一段 HTML 并文字解释如何写 CSS |
| ❌ | baseline | ON | SAE 注入对思考模式的影响微弱可忽略 |

### 关键发现

1. **V1（L15#1511 / L26#449 / L16#1298）是真正起作用的核心三元组**
   - 加入 V2 的 L15#7439 / L26#12083 / L27#2890 没有可见增益 → 这三个是 n=1 单 QA 的 false positive
   - 加入 full 的 L17/L34 一组也没有独立增益

2. **L16#1298 单点最强**
   - 仅它一个就能解锁 HTML+CSS+JS 三件套（其他配置经常只给 HTML+CSS）
   - 假设：它压制的是 Gemma 的「写代码时省略 JS / 用单一 HTML 文件糊弄」倾向

3. **Think 模式与 SAE 注入冲突**
   - 跨所有配置：ON 一律比 OFF 差，且经常严重崩坏
   - 推测机制：`<|think|>` 进入「思维段」分布，clamp 对思维 token 的常数偏移把推理段扯坏，再污染答案段
   - 实操结论：**clamp 与 Think 不要同时开**

4. **「早期 GPT 页面」知识缺失（已确认）**
   - 续写 prompt: "The official ChatGPT GPT-3.5 era webpage used a background color of"
   - Gemma 自述无该具体知识
   - V1 之所以能给出最接近版本，是因为它激活了**结构先验**（sidebar / 多区域 / 三栏），不是恢复了视觉知识
   - 即：**没有「早期 GPT 页面知识神经元」可挖**，看到的提升是「通用前端复杂度」轴的副作用
